# Lamark Pilot Go/No-Go Risk Acceptance

## Purpose

This document defines risk acceptance requirements for a future Lamark pilot Go/No-Go decision.

## Risk Acceptance Criteria

| Area | Required Record | Gate Impact |
|---|---|---|
| Known risks | Risks are listed and categorized | Required |
| Risk owner | Each accepted risk has a human owner | Required |
| Risk rationale | Acceptance rationale is documented | Required |
| Conditions | Conditions or constraints are recorded | Required |
| Blocked risks | Unacceptable risks produce no-go | Blocking |
| Execution limit | Risk acceptance does not execute the pilot | Blocking |

## Acceptance Rule

Risk acceptance must be explicit, human-owned, bounded, and documentary.

## Prohibited Acceptance

Risk acceptance may not approve production activation, real customer use, sensitive financial data use, real agents, real integrations, permissions, secrets, dependencies, CI, runners, databases, services, or runtime execution.
