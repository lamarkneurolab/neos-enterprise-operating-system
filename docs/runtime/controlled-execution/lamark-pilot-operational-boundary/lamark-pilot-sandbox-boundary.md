# Lamark Pilot Sandbox Boundary

## Purpose

This document defines the sandbox boundary for the Lamark controlled pilot.

## Sandbox Definition

The sandbox is a constrained documentary environment for knowledge management, document classification planning, evidence registry planning, audit requirement planning, and controlled pilot readiness review.

## Sandbox Allowed Activities

- Classify approved sandbox copies.
- Build knowledge management categories.
- Define evidence capture requirements.
- Define audit event requirements.
- Define authorization checkpoints.
- Define rollback and incident boundaries.
- Prepare handoff requirements.

## Sandbox Exclusions

The sandbox excludes production activation, real customer workflows, sensitive financial data, external commercial diagnosis, autonomous document movement, real document deletion, real document sharing, information sending, permission modification, real document transformation, external integration creation, agents, runners, CI, databases, and services.

## Exit Rule

Leaving the sandbox requires explicit human authorization, evidence package review, audit correlation, and a future separately authorized action.
