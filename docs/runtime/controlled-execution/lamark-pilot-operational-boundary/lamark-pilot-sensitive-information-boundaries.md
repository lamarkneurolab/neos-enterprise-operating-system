# Lamark Pilot Sensitive Information Boundaries

## Purpose

This document defines sensitive information boundaries for the Lamark controlled pilot.

## Sensitive Information Categories

| Category | Pilot Status |
|---|---|
| Sensitive financial data | Blocked |
| Customer personal data | Blocked |
| Customer operational data | Blocked |
| Secrets and credentials | Blocked |
| Tokens and API keys | Blocked |
| Bank, billing, or tax records | Blocked |
| External commercial diagnosis data | Blocked |
| Production logs with sensitive content | Blocked |
| Permission and access control secrets | Blocked |

## Allowed Information Categories

The pilot may use non-sensitive sandbox content, approved non-sensitive metadata, documentary governance records, evidence records, audit requirement records, and knowledge management taxonomy notes.

## Review Rules

Any suspected sensitive information must be treated as blocked until a human reviewer classifies it and provides explicit scoped authorization for a permitted handling action.
