# Lamark Pilot Boundary Overview

## Purpose

This document defines the documentary operating boundary for Lamark Pilot Operational Boundary Matrix v0.1.0.

Lamark is the first controlled pilot environment for NEOS. It is not open production. The pilot begins with limits, sandbox, evidence, audit, and human authorization before any operational expansion.

## Scope

The boundary covers what NEOS may do, what NEOS must not do, which actions require human authorization, which actions remain blocked, what data may be consulted, what data must not be consulted, what documents may be classified, what documents must not be modified, what evidence must be recorded, what events must be audited, what rollback conditions apply, what incidents block execution, and what handoffs are required.

## Core Pilot Rules

- Lamark is a controlled pilot, not open production.
- No external sales.
- No real customers.
- No sensitive financial data.
- No external commercial diagnosis.
- No autonomous action without authorization.
- No moving real documents without authorization.
- No deleting real documents without authorization.
- No sharing real documents without authorization.
- No sending information without authorization.
- No permission changes without authorization.
- No transforming real documents without authorization.
- First limits, sandbox, evidence, and authorization.

## Non-Execution Boundary

This boundary matrix does not execute runtime, activate production, create agents, create external integrations, create runners, create CI, create databases, create services, modify permissions, manage secrets, install dependencies, delete files, push branches, open PRs, or merge changes.

## Relationship With Blocks 13 Through 19

Blocks 13 through 18 define controlled execution protocol, monitoring, closure, readiness, release decision governance, and authorization traceability. Block 19 defines evidence registry and audit correlation. Block 20 applies those governance layers to the Lamark controlled pilot boundary.
