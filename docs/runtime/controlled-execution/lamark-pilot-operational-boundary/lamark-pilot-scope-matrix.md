# Lamark Pilot Scope Matrix

## Purpose

This document defines the operational scope matrix for the controlled Lamark pilot.

## Scope Matrix

| Area | Allowed Without Additional Authorization | Requires Human Authorization | Blocked |
|---|---|---|---|
| Knowledge management | Sandbox classification, taxonomy review, evidence mapping | Use of real Lamark documents | External publishing or autonomous document movement |
| Document classification | Classification of approved sandbox copies | Classification of real documents | Modification, deletion, sharing, or transformation of real documents |
| Evidence registry | Documentary evidence records and traceability | Evidence use tied to a future controlled action | Evidence as authorization |
| Audit | Audit requirement definition and audit event correlation | Audit review for real pilot activity | Audit as permission to execute |
| Data access | Non-sensitive sandbox data and approved metadata | Limited read-only real document metadata | Sensitive financial data or customer data |
| Runtime activity | No real runtime | Future separately authorized dry-run or controlled action | Production activation |
| External operations | None | Future separately authorized stakeholder communication | Sales, real customer workflows, or external diagnosis |
| Permissions | Review of boundary rules | Future separately authorized permission review | Permission modification |

## Interpretation Rules

Allowed items remain documentary and sandbox-bound. Human authorization must be explicit, current, scoped, and tied to the specific action. Blocked items remain blocked until a future separately authorized action supersedes the block.
