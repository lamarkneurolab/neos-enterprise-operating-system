# Lamark Pilot Allowed Actions

## Purpose

This document defines actions allowed inside the controlled Lamark pilot without expanding into runtime execution or production.

## Allowed Documentary Actions

- Define operational boundaries.
- Define sandbox rules.
- Define evidence requirements.
- Define audit requirements.
- Define human authorization requirements.
- Define rollback and incident boundaries.
- Define handoff requirements.
- Classify approved sandbox documents.
- Map approved knowledge management categories.
- Register documentary evidence for approved pilot planning.
- Correlate audit requirements with evidence records.
- Review non-sensitive metadata explicitly approved for pilot planning.

## Conditions

Allowed actions must remain read-only, documentary, sandbox-bound, and non-destructive unless a future explicit human authorization states otherwise.

## Non-Authorization Rule

An allowed documentary action does not authorize runtime execution, production activation, external integration, agent creation, real document movement, real document deletion, real document sharing, information sending, permission modification, or document transformation.
