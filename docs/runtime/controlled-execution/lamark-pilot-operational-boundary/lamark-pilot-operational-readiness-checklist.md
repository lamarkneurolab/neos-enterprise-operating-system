# Lamark Pilot Operational Readiness Checklist

## Purpose

This checklist defines documentary readiness criteria for the Lamark controlled pilot.

## Readiness Checklist

| Item | Required Status |
|---|---|
| Lamark identified as controlled pilot, not open production | Required |
| Allowed actions documented | Required |
| Blocked actions documented | Required |
| Human authorization requirements documented | Required |
| Data access boundaries documented | Required |
| Document handling boundaries documented | Required |
| Sensitive information boundaries documented | Required |
| Sandbox boundary documented | Required |
| Evidence requirements documented | Required |
| Audit requirements documented | Required |
| Rollback boundaries documented | Required |
| Incident boundaries documented | Required |
| Handoff requirements documented | Required |
| No external sales | Required |
| No real customers | Required |
| No sensitive financial data | Required |
| No external commercial diagnosis | Required |
| No autonomous action without authorization | Required |
| No real document movement without authorization | Required |
| No real document deletion without authorization | Required |
| No real document sharing without authorization | Required |
| No information sending without authorization | Required |
| No permission modification without authorization | Required |
| No real document transformation without authorization | Required |

## Readiness Rule

Operational readiness is documentary only. It does not authorize runtime execution, production activation, external integrations, agents, services, databases, CI, runners, permissions changes, secrets handling, or merge.
