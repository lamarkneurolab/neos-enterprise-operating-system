## Core Rules

- Evidence records do not authorize execution.
- Passing checks are validation evidence, not release authorization.
- Historical evidence is not current authorization.
- Audit correlation is traceability, not permission.
- Evidence correlaciona.
- Auditoria verifica.
- Nada ejecuta.
- Execution remains blocked until a future separately authorized execution action exists.

## Lamark Pilot Acceptance Boundary

Lamark is accepted only as the first controlled pilot environment. It is not open production. Initial activity must stay focused on knowledge management, document classification, evidence, audit, and knowledge base foundations. No external sales, real customer activity, sensitive financial data handling, external commercial diagnosis, autonomous action, or real document movement is authorized by this checklist.

## Restrictions Confirmation

This checklist confirms documentary scope only. It does not authorize runtime execution, production activation, agents, external integrations, runners, CI, databases, services, permission changes, secrets, dependencies, file deletion, destructive changes, merge, future authorization, or reuse of the PR #16 authorization.
