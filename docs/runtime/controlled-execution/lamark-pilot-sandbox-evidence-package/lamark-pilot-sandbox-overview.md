# Lamark Pilot Sandbox Overview

## Purpose

This document defines the Lamark pilot sandbox preparation layer for Lamark Pilot Sandbox Preparation & Evidence Package v0.1.0.

## Sandbox Definition

The Lamark sandbox is a documentary sandbox only. It is not a production environment, not an open customer environment, and not an authorization to execute runtime behavior.

The sandbox exists to prepare controlled pilot evidence, document-copy rules, data rules, audit correlation, readiness controls, rollback triggers, incident blockers, and handoff requirements before any future operational action is considered.

## Core Rules

- Lamark is a controlled pilot, not open production.
- The sandbox is documentary, not productive.
- No external sales.
- No real customers.
- No sensitive financial data.
- No external commercial diagnosis.
- No autonomous action without human authorization.
- No real document movement, deletion, sharing, permission change, information sending, or transformation without authorization.
- Only authorized sandbox copies may be used.
- First sandbox, evidence, and authorization.

## Non-Execution Boundary

This package does not execute runtime, activate production, create agents, create external integrations, create runners, create CI, create databases, create services, modify permissions, manage secrets, install dependencies, delete files, touch real documents, open PRs, or merge changes.
