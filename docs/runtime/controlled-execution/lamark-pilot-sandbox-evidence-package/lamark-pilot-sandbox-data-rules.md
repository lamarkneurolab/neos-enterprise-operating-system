# Lamark Pilot Sandbox Data Rules

## Purpose

This document defines data allowed in the Lamark documentary sandbox.

## Allowed Data

The sandbox may use:

- Synthetic examples.
- Redacted examples.
- Approved metadata.
- Approved taxonomy labels.
- Approved document category names.
- Evidence identifiers.
- Audit event identifiers.
- Authorization references.
- Readiness checklist statuses.
- Handoff status notes.

## Data Conditions

Allowed data must meet all of the following conditions:

- No sensitive financial data.
- No real customer data.
- No credentials, secrets, or tokens.
- No unredacted personal data.
- No external commercial diagnosis.
- No data that would require production access.
- No data that would require permission changes.

## Data Escalation Rule

If data sensitivity is unclear, the data must be treated as blocked until human review confirms that it is allowed for sandbox use.
