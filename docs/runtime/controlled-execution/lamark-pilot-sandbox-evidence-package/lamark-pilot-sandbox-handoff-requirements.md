# Lamark Pilot Sandbox Handoff Requirements

## Purpose

This document defines handoff requirements for Lamark sandbox preparation.

## Required Handoff Contents

A handoff must include:

- Sandbox scope summary.
- Inclusion and exclusion summary.
- Authorized-copy status.
- Allowed-data status.
- Sensitive-data blocklist confirmation.
- Human authorization status.
- Evidence package index.
- Audit-correlation status.
- Readiness control status.
- Rollback trigger status.
- Incident blocker status.
- Open issues.
- Explicit statement that execution remains blocked.

## Handoff Boundary

The handoff transfers documentary context only. It does not authorize production activation, runtime execution, external sales, real customer activity, data expansion, permission changes, or autonomous action.

## Acceptance Rule

The receiving reviewer must confirm that the handoff package is complete, correlated, and still within the documentary sandbox boundary.
