# Lamark Pilot Dry Authorization Package

## Purpose

This document defines the dry authorization package for a future controlled Lamark pilot.

## Dry Authorization Definition

A dry authorization package is a documentary collection of required authorization evidence, decision records, blockers, and control confirmations prepared before any real execution request.

It is not a production activation, not a runtime approval, not a customer-facing approval, and not permission to operate agents or integrations.

## Required Package Contents

- Final evidence review.
- Final sandbox review.
- Final data boundary review.
- Final document handling review.
- Final human approval review.
- Final execution window review.
- Final incident blocker review.
- Final rollback review.
- Final audit correlation review.
- Final handoff review.
- Final no-go review.
- Non-execution confirmation.
- Pre-execution decision record.
- Final checklist.

## Authorization Limits

The package may support a future authorization request, but it cannot grant authorization by itself.

Any future execution requires a separate explicit approval naming the branch, base SHA, head SHA, commit, scope, execution window, evidence package, rollback boundary, incident boundary, monitoring boundary, and human authorizer.
