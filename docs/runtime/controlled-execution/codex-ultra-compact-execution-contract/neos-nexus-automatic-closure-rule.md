# NEOS/NEXUS Automatic Closure Rule v0.1.0

## Closure Trigger

When closing any NEOS/NEXUS block or process, the assistant MUST automatically generate the closure package.

## Required Closure Package

The closure package MUST include:

1. Ultra compact summary.
2. Updated canonical state.
3. Continuity instructions.
4. Downloadable Markdown file.
5. Prompt for a new chat.

## Closure Discipline

The closure package MUST be concise, operational, and sufficient for continuity without reopening prior context.

The closure package MUST preserve the current canonical state and identify the next authorized step without implying authorization for any future pull request, merge, runtime action, external impact, or real automation.

## Exclusions

Automatic closure MUST NOT include production actions, secrets handling, permission changes, CI/runner changes, dependency changes, database changes, external service operations, real Drive operations, real communications, real automations, C4-C7 data handling, deletions, destructive changes, runtime execution, real agent activation, merge, or auto-merge.
