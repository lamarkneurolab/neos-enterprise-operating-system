# Live Rollback Trigger Conditions

## Purpose

Define rollback triggers during future controlled monitoring.

## Trigger conditions

- Critical drift.
- Critical anomaly.
- Stop condition reached.
- Emergency pause cannot be resolved.
- Evidence or audit becomes invalid.
- Rollback readiness changes.
- Safety signal requires rollback.
- Human reviewer decides rollback is required.

## Rollback prerequisites

- Approved rollback plan.
- Evidence reference.
- Audit reference.
- Human escalation reference when required.
- Impact review.
- Completion criteria.

## Non-rollback rule

This document does not execute rollback. It defines when rollback must be
considered, evidenced, audited and escalated.
