# Post-Start Observation Checklist

## Purpose

Define the checklist required immediately after a future controlled execution
start.

## Checklist

- [ ] Activation session is still valid.
- [ ] Human go/no-go remains exact and current.
- [ ] PR, branch, head SHA and commit match authorization.
- [ ] Scope and action remain locked.
- [ ] Runtime window is active.
- [ ] Monitoring boundary is active.
- [ ] Health signals are visible.
- [ ] Safety signals are visible.
- [ ] Evidence capture is active.
- [ ] Audit linkage is active.
- [ ] Drift detection is active.
- [ ] Anomaly handling path is available.
- [ ] Stop conditions are known.
- [ ] Emergency pause path is available.
- [ ] Human escalation path is available.
- [ ] Rollback trigger conditions are known.

## Blocking rule

Any unchecked item requires hold, stop, emergency pause or human escalation
review.
